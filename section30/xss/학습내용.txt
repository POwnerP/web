[xss공격]
: 사용자의 요청이 전송되어 웹사이트에 표시되거나 하는 경우에 잘 사용됨.

예를 들어 유저가 댓글을 달고 그 댓글이 바로 갱신되는 사이트가 있다면,
그 내용이 사이트의 html파일에 추가되어 화면에 표시되는 방식이기 때문에 
댓글에 script 내용을 작성하면  웹사이트를 다양한 방식으로 조작할 수 있음.

1. 방어하는방법
1) 사용자로부터 입력받는 텍스트는 파싱되지 않고 텍스트로만 처리되게 하기.(데이터를 원시텍스트로 저장, 텍스트로만 인식되게하기.)(excape 라고 함.)
 - 사용자가 입력한 데이터를 불러오는 경우에는 ejs 태그를 '<%='을 사용하기. '<%='은 결국 불러온 데이터를 텍스트로만 나타나게 함.
   '<%-'에 포함된 내용은 html 문장으로도  해석될 수 있음.
 - 밑의 방법보다 우선되는 방법임. 이 방법이 안된다면 '2)' 방법을 사용.
2) excape하지 않은상황에서는 의심되는 문장을 선별하여 제거하는 코드를 작성하는 방법이 있음.(sanitize 라고 함.)
 - xss 라는 패키지를 다운받음
   npm install xss

   * out of the box: 별개로 조치(파일 설치 등..)를 하지 않아도 문제없이 처리되는.

[기본적으로 조심해야할 것]
1. 중요한 폴더를 static으로 지정하지 말것.
 - static으로 지정한 폴더는 localhost:3000/basic.css 와 같이 직접 url을 입력하여 파일 내부를 확인할 수 있음. - css는 괜찮지만 고객 정보 등이 들어있는 파일이라면 위험함.
2. 날것의 오류페이지를 되도록 보여주지 말것.
 - 에러페이지의 처리를 통해 날것의 오류페이지를 보여주지 말것.
